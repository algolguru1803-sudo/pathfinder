# ADR-0011: Таймлайн «Трейсинг» Вариант C — обзор по токенам, фоновые полосы фаз, нормализация роли

- **Дата:** 2026-06-13
- **Статус:** принято
- **Задача:** rework-agent-timeline

## Контекст

Старый таймлайн «Трейсинг» (`renderGantt(subs)` в `templates/dashboard.html`) переставал читаться на
длинном прогоне: десятки под-агентов × часы сжимались в ширину карты, ось состояла из двух меток
(`начало`/`конец`), `fmtDur` не умел часы (на 4-часовом окне выдавал «247м»), а цвет почти всех баров был
**хеш-цветом**, а не палитрой ролей (см. баг палитры ниже). Источник правды `/trace` (`subs[]`) **уже
содержит всё нужное** (`startTs/endTs`, `durationMs`, `out`, `phase`, `role`, `summary`, `spanId`, `ok`,
`model`, `contextPct`, `costUsd`) — серверная модель практически не требует изменений. На гейте человек
выбрал **Вариант C** (обзор-целиком + brush-окно + детальная зона) и добавил два требования: обзор рисовать
**по производству токенов** (не по числу агентов) и **разбить график на фазы workflow**.

## Решение

Переписать `renderGantt(subs, tr)` под Вариант C **целиком на клиенте, без правок сервера**. Три ключевых,
неочевидных решения внутри:

### 1. Обзорный график — по производству токенов во времени (token-rate area chart)

Высота верхней площадной диаграммы = **темп токенов** в момент времени, а не плотность параллелизма.
Поскольку `out` под-агента — это **итог за весь его интервал** (а не временной ряд), токены раздаются как
**скорость**: для каждого активного интервала `rate = out / max(1, durationMs)` (ток/мс); шкала `[T0..T1]`
дискретизируется на ~240 бакетов (`bw = SPAN/N`), и каждый интервал прибавляет свой `rate` во все бакеты,
где он активен (`[s..e]`). Высота бакета = `tokRate[b] / tokMax`. Интервал с `out==null|≤0` вклада не
вносит (не падать); микро-интервал (0 длительности) → его токены целиком в его бакет (`dur=max(1,e-s)`,
без деления на ноль). Суммарные токены окна ≈ `Σ tokRate[b]*bw ≈ Σ out`.

### 2. График разбит на фазы workflow (фоновые регионы + «вне фазы»)

Фазы (`phase`: EXPLORE/ELABORATE/IMPLEMENT/VERIFY + бакет **«вне фазы»** для `null`) рисуются как
**фоновые регионы по оси времени** — полоса-заголовки над token-графиком в обзоре и фон за барами
(`z-index` ниже баров) + заголовки в детальной зоне. Диапазон фазы = `[min startTs … max endTs]` её
интервалов; порядок — `PHASE_ORDER` (EXPLORE→ELABORATE→IMPLEMENT→VERIFY→вне фазы), показываются только
присутствующие. **Цвет БАРОВ остаётся по роли** — фаза только фон/полоса (гибрид: фон=фаза, заливка=роль).

### 3. Нормализация роли (`roleKey`) — фикс корневого бага палитры

Цвет бара = роль через `roleColor(roleKey(role))`. `roleKey(r)` срезает namespace-префикс
(`r.split(":").pop()`) и `wf-` → нормализованный ключ; ключи `ROLE_COLORS` приведены к этой же форме
(`coder/explorer/planner/reviewer/documenter/оркестратор`) + добавлены `Explore` (haiku-ревьюеры) и
`general-purpose`. `roleLabel` даёт короткую подпись (`general-purpose`→`general`).

## Почему

- **Темп токенов честнее показывает «где сожгли работу», чем плотность агентов.** Прямое требование
  человека. Один долгий агент может произвести больше токенов, чем три коротких параллельных — счёт по
  числу баров это скрывает. Все поля (`out`, `durationMs`) уже в `subs[]` — сервер не трогаем.
- **Фазы дают ось «в какой стадии шёл прогон и сколько в ней работы».** Прямое требование человека; `phase`
  уже в `subs[]`. Цвет бара оставлен за ролью, т.к. роль и фаза — две **ортогональные** шкалы; смешать их в
  одном канале (цвет) — потерять одну. Отсюда же две легенды (роли + фазы) над таймлайном.
- **Нормализация роли — это устранение корневого бага, а не косметика.** Роль приходит как
  `ai-pathfinder:wf-coder` (namespace плагина), `ROLE_COLORS` промахивался → почти все бары хеш-цветные,
  легенда нечитаема. Нормализация — минимальная правка, разблокирующая читаемую палитру и легенду.
- **Вариант C даёт обзор-целиком + зум окном без горизонтального скролла.** Обзор всегда = весь прогон;
  brush выбирает окно; детальная зона «вмещает окно в ширину карты» → бары в окне всегда читаемы. Это
  снимает вопрос масштаба (q3) без отдельного тумблера «уместить/фикс.».
- **0 правок сервера = нет перезапуска `scripts/server.py` (:8480) и риска, что дашборд крутит python из
  кэша плагина** (память проекта). `build_trace` читает `telemetry.jsonl` целиком (кэш 3 с) — не утяжелять.

## Честные оговорки (зафиксированы в коде-комментариях)

- **Token-rate — аппроксимация.** `out` — итог за интервал, не временной ряд; равномерное распределение по
  длительности (rate) — допущение, реальный темп внутри интервала неизвестен. `Σ` по бакетам ≈ `Σ out` с
  точностью до дискретизации/нахлёста бакетов. Это **оценка темпа**, не точный профиль.
- **`phase` разрежён — и это легитимно, не прятать.** `phase` под-агента берётся из `state.json.phase`
  оркестратора **на момент `subagent.start`** (`context_from_state` → `{phase, iteration}` в
  `subagent.start`, `scripts/telemetry_hook.py`). Значит фаза заполнена **только** если сабагент запущен при
  непустом `state.json.phase`. В тест-датасете (`sample-trace.json`) у **26 из 36 записей `phase=null`** →
  «вне фазы» — крупная легитимная полоса (много `general-purpose`/legacy-агентов). На нормальном
  воркфлоу-прогоне фазы заполнены. Отдельно: **`workstream` хук НЕ пишет вообще** — `context_from_state`
  возвращает только `{phase, iteration}`, в `subagent.start` поля `workstream` нет, в `subs[]` оно
  **всегда `null`**. Старая подпись бара «role · workstream» фактически была = только role.
- **Фазы могут перекрываться** (агент фазы X завершился позже старта фазы Y). Выбран **читаемый вариант:
  рисуем по фактическим границам, нахлёст допускаем** (заголовки фаз — отдельная строка; фон в детальной
  зоне может накладываться — берём по порядку `PHASE_ORDER`). Альтернатива (непересекающиеся диапазоны по
  медиане) сложнее и теряет фактические границы — отклонена.

## Подводный камень: `sig`-гард перерисовки `#trace-summary`

`renderTraceSummary` (`templates/dashboard.html`) пропускает перерисовку, если контентная сигнатура `sig`
не изменилась — иначе тик `traceTick` (опрос `/trace` каждые ~3 с) **затирал бы** интерактивное состояние
(brush-окно, скролл, раскрытие) при каждом ответе сервера. Поэтому интерактив таймлайна обязан быть **либо
CSS-only** (`:hover`, `<details>`), **либо жить вне summary-DOM**:

- **тултип** — отдельный `position:fixed` контейнер вне `#trace-summary` (по образцу `#trace-feed`,
  отделённого в `ensureTraceShell`);
- **brush-состояние `bs/be`** — модульные переменные вне DOM;
- **состояние кнопок режима/зума** (если появятся) — модульная переменная **+ обязательно в `sig`**, чтобы
  тик не перетирал, но перерисовывал при переключении;
- **подсветка карточки агента по клику на бар** — CSS-класс на карточке, снимаемый по таймауту, без участия
  в `sig`.

## Последствия

- **Сервер `/trace` и `_agent_record` не меняются.** Воркстрим серверных полей (`iteration`, явный
  `running`) оставлен пустым по умолчанию (`status:running` выводится на клиенте из `endTs==null`). Любое
  будущее поле для таймлайна — дешёвой дозаписью в `_agent_record` из уже распарсенных событий, **не** новым
  проходом транскриптов (`build_trace` читает файл целиком — горячее место).
- **Новый код целиком в `renderGantt`** — кто правит таймлайн, читает `subs[]`/`tr.timeline` и обязан
  держать интерактив вне `sig`-перерисовки (см. выше). Микро-интервалы → точка-маркер; `span-`-плейсхолдеры
  (`startTs==null`) → не баром, а сноской.
- **`running` почти не встречается на завершённом прогоне** (`build_trace` — посмертная модель); ценность
  ветки — для живого наблюдения.
- **Две темы обязательны** — все цвета фона/линий только через `var(--…)`; ролевые цвета остаются
  хардкод-палитрой `ROLE_COLORS` (как раньше), тинты фаз — токены `--ph-*` для обеих тем.

## Альтернативы (отклонено)

- **Обзор по числу параллельных агентов** — отклонён человеком: скрывает, где реально сожжены токены/работа.
- **Цвет бара по фазе** (а не по роли) — отклонён: фаза вынесена в фон/полосы, цвет оставлен за ролью; иначе
  одна из двух ортогональных шкал теряется.
- **Серверный флаг `status:"running"` / `workstream`** — отклонён: дублирует клиентскую логику
  (`endTs==null`), требует перезапуска сервера; `workstream` хук не пишет в принципе.
- **Непересекающиеся диапазоны фаз по медиане** — сложнее, теряет фактические границы; выбран нахлёст.
- **Горизонтальный скролл/фикс-масштаб с тумблером** (наследие вариантов B) — заменён связкой
  обзор+brush+детальная зона; отдельный тумблер не нужен.
